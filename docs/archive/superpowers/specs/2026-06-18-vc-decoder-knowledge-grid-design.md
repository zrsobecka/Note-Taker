# VC Decoder Knowledge Grid Design Archive

Archived one-off design for adding a navigation layer to the Obsidian `VC Decoder` folder.

## Goal

Create additive navigation without deleting, moving, or rewriting existing lesson notes.

## Planned Outputs

- `VC Decoder/00 VC Decoder Map.md`
- `VC Decoder/Maps/*.md`
- `VC Decoder/Concepts/*.md`

## Structure

- Main map groups the course into: Startup Logic, Fund Math, Investor Thinking, Stage Map, Investment Case, Fundraising Materials, Fundraising Process, Terms & Deal Structure.
- Thematic maps include short area description, key lessons, key concepts, learning order, and "if you do not understand X, return to Y".
- Concept notes explain one term with: meaning, VC importance, investor view, simple example, related lessons, related concepts.

## Safety Rules

- Do not delete existing notes.
- Do not move existing notes.
- Do not rewrite lessons from scratch.
- Add navigation only where useful.
- Do not overwrite files without reading them first.
- Prefer fewer useful links over mass linking.

## Verification

Check that maps and concepts exist, link to real lessons, keep consistent structure, and do not damage original lesson content.
