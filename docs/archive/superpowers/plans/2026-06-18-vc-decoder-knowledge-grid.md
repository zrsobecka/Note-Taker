# VC Decoder Knowledge Grid Plan Archive

Archived one-off implementation plan for building an additive Obsidian knowledge grid for `VC Decoder`.

## Goal

Keep lesson notes as source material and add map/concept navigation around them.

## Architecture

- Existing lessons stay in place.
- New maps live in `VC Decoder/Maps/`.
- New concept notes live in `VC Decoder/Concepts/`.
- Existing lessons may receive only a small appended navigation section.

## Tasks

1. Inventory `VC Decoder` and check whether target folders/files already exist.
2. Create `VC Decoder/00 VC Decoder Map.md`.
3. Create eight thematic maps:
   - Startup Logic
   - Fund Math
   - Investor Thinking
   - Stage Map
   - Investment Case
   - Fundraising Materials
   - Fundraising Process
   - Terms & Deal Structure
4. Create priority concept notes for startup, VC, fundraising, fund math, and deal-structure terms.
5. Append minimal navigation to existing lessons only where useful.
6. Verify folders, files, links, concept structure, and original lesson preservation.

## Concept Note Template

```markdown
# [Concept]

## Co to znaczy
## Dlaczego ważne w VC
## Jak inwestor na to patrzy
## Prosty przykład
## Powiązane lekcje
## Powiązane pojęcia
```
